Gaining Initial Foothold

Understanding the CVE

Wing FTP Server versions prior to 7.4.4 are vulnerable to an unauthenticated remote code execution (RCE) flaw (CVE-2025-47812). This vulnerability arises from improper handling of NULL bytes in the ‘username’ parameter during login, leading to Lua code injection into session files. These maliciously crafted session files are subsequently executed when authenticated functionalities (e.g., /dir.html) are accessed, resulting in arbitrary command execution on the server with elevated privileges (root on Linux, SYSTEM on Windows)

Getting into User account

The reverse shell got us into the machine as wingftp user, but we need to escalate our privilege into user account. Looking into the /home directory we see that there exists a user named wacky but we don’t have permission to view that user directory. Our working directory is /opt/wftpserver. Lets see if there are any interesting files what might have maybe exposed user details.

Finding the credentials

The user accounts of the wftp appears to be saved in the /opt/wftpserver/Data/1/users/ directory where we can see our user’s file wacky.xml as well.
