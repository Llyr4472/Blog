THM Easy Room

  • Challenge: https://tryhackme.com/room/investigatingwindows
  • Background:

    A windows machine has been hacked, its your job to go investigate this windows machine and find clues to what the hacker might have done.

  • easy
  • 45 min

Questions

  1. What’s the version and year of the windows machine?
    -> Windows Server 2016

Just went to settings and about PC to find the answer

  1. Which user logged in last?
    -> Administrator

Went to Event Viewer, then inside Windows Logs > Security logs. Filter by event ID 4624

  1. When did John log onto the system last?
    -> 03/02/2019 5:48:32 PM

Use cmd to find user details.

  1. What IP does the system connect to when it first starts?
    -> 10.34.2.3

To see startup task we need to look at HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows > CurrentVersion > Run, which contains startup instructions. There is following key there which contains required ip address.

  1. What two accounts had administrative privileges (other than the Administrator user)?
    -> Guest, Jenny

We can use following command to see all administrators.

  1. Whats the name of the scheduled task that is malicous.
    -> Clean file system

Opening Task Scheduler to view scheduled task, we can see following tasks. Looking through them the clean file system is running a suspicious powershell script.

  1. What file was the task trying to run daily?
    -> nc.ps1

  1. What port did this file listen locally for?
    -> 1348

  2. When did Jenny last logon?
    -> never

Used following cmd command to see Jenny account details

  1. At what date did the compromise take place?
    -> 03/02/2019

Looking back, the Jenny password was changed and the account was also probably created on this date and the malicious scheduled task was created on this day.

  1. During the compromise, at what time did Windows first assign special privileges to a new logon?
    -> 03/02/2019 4:04:49 PM

We know the compromise date so lets look at security events of that day. Lets filter the logs further using the Event ID 4672 which logs any special privileges assigned to a new logon. After filtering we can see following event in which loads of new perms were added to a account.

  1. What tool was used to get Windows passwords?
    -> mimikatz

Lets look at that TMP folder containing nc.ps1 file to see if it contains any clue. There are several files there. Looking in the txt files for clue we see that mim-out file is output of mimikatz. So, we can conclude that it was the tool used.

  1. What was the attackers external control and command servers IP?
    -> 76.32.97.132

Looking at the files in TMP folder, there was nothing useful. Lets look at hosts file to see they used dns poisoning to redirect legit site to C2. Opening C:\Windows\System32\drivers\etc\hosts that seems to be the case. The google.com domain has been redirected to a malicious ip.

  1. What was the extension name of the shell uploaded via the servers website?
    -> .jsp

Going to the C:\inetpub\wwwroot directory to see the files uploaded to website. And there are two files with .jsp extension. jsp seems to be some kind of java program for servers The gif file seems inherently harmless even if named shell.

  1. What was the last port the attacker opened?
    -> 1337

To see the opened port, we need to check windows firewall. Looking in Inbound Rules since we are looking for rules for incoming traffic. The first rule straight-up seems sus. Looking its details shows us that 1337 port was opened.

  1. Check for DNS poisoning, what site was targeted?
    -> google.com

We already found this while looking for C2 server.
